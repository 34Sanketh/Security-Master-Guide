01
Make All Repos Private on GitHub
Required
⌄

Do this for each repo one by one. Takes about 5 minutes total.

1
Open github.com in your browser
2
Tap the ☰ menu → tap Your repositories
3
Find and tap your repo name
4
Tap the Settings tab (scroll the tab row to find it)
5
Scroll all the way down to the red Danger Zone section
6
Tap Change visibility → tap Make private
7
A box appears — type the repo name exactly as shown
8
Tap "I understand, change repository visibility"
🔁 Repeat these exact steps for each of your other repos
✓All repos private = your source code is now hidden from the public internet.

06
Obfuscate Your JavaScript Code
Security
⌄

Makes your JS logic unreadable to anyone who views the page source. Only obfuscate JS — HTML and CSS don't need it.

ℹYour HTML file has 3 parts: HTML (between body tags), CSS (between style tags), JS (between script tags). We only touch the JS part.
1
Open your HTML file on GitHub
2
Find the <script> tag near the bottom — copy everything between the opening and closing script tags (not the tags themselves)
3
Open obfuscator.io in a new tab — paste your JS code in the left box
4
Apply these settings (see table below) → tap Obfuscate
5
Copy all obfuscated code from the right box
6
Go back to your HTML file on GitHub → tap the pencil icon to edit
7
Delete the original JS code between the script tags — paste the obfuscated code in its place
8
Scroll down → tap Commit changes — Netlify auto-redeploys in 1–2 minutes

07
Add Firebase Auth to Unprotected Sites
Security
⌄

For any site that doesn't already have Google login, add it using the code below. Edit your HTML file on GitHub.

Step 1 — Add Firebase scripts inside <head>

HTML
<!-- Firebase -->
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-app-compat.js"></script>
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-auth-compat.js"></script>

Step 2 — Add Auth logic at the top of your <script>

JavaScript
// Copy your Firebase config from your existing project
const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
};
firebase.initializeApp(firebaseConfig);

// Auth check — runs before anything else
firebase.auth().onAuthStateChanged(function(user) {
  if (user) {
    document.getElementById('app').style.display = 'block';
    document.getElementById('loginScreen').style.display = 'none';
  } else {
    document.getElementById('app').style.display = 'none';
    document.getElementById('loginScreen').style.display = 'flex';
  }
});

function googleLogin() {
  const provider = new firebase.auth.GoogleAuthProvider();
  firebase.auth().signInWithPopup(provider);
}
function logout() {
  firebase.auth().signOut();
}

Step 3 — Wrap your HTML content in login/app divs

HTML
<!-- LOGIN SCREEN -->
<div id="loginScreen" style="display:flex;flex-direction:column;
  align-items:center;justify-content:center;height:100vh;">
  <h1>Your App Name</h1>
  <p>Sign in to continue</p>
  <button onclick="googleLogin()">Sign in with Google</button>
</div>

<!-- MAIN APP (hidden until login) -->
<div id="app" style="display:none">
  <!-- ALL your existing HTML goes here -->
</div>
08
Tighten Firestore Security Rules
Security
⌄

Ensures each logged-in user can only read and write their own data — nobody else's.

1
Open console.firebase.google.com → select your project
2
Left menu → Firestore Database → tap Rules tab
3
Replace everything with the rules below
4
Tap Publish
Firestore Rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}
✓
✅ Logged-in users can only access their own data
❌ No one can read another user's data
❌ Logged-out visitors cannot read anything

⚠️ Important Reminders

1
Auto-deploy is active. Every time you edit your HTML and push to GitHub, Netlify redeploys automatically. No extra steps needed.
2
Obfuscate last. Always finish all edits first, then obfuscate. Never obfuscate before you're done — obfuscated code is very hard to edit.
3
Keep a master copy. Save your original, un-obfuscated HTML file somewhere safe (e.g. Google Drive). This is the copy you edit. The GitHub version is the published/obfuscated one.
4
Firebase free plan is sufficient for personal use. No need to upgrade unless you hit usage limits.
